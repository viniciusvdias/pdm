# Seminars

Due to the shift of the course toward a more practical approach, many of these topics do not have scientific papers as starting points, but I believe technical seminars on them would be interesting. However, I still expect a technical/scientific focus on the topics, not just a "marketing" talk like many we often see. I expect a clear description of the involved concepts, a critical analysis of pros and cons, etc.  

Below is a list of topics. Each graduate/isolated enrollment student must choose a topic and write their name at the end of the line to reserve it. The presentation dates will be indicated on each line once they are defined.  

1. **TensorFlow** – Distributed machine learning framework, dataflow graphs, scalable training and inference.
- [Documentation](https://www.tensorflow.org/?hl=pt-br) [Paper](https://www.usenix.org/system/files/conference/osdi16/osdi16-abadi.pdf)
2. **Cassandra** (important databases in big data, the focus should be on their implementation, as discussed in the papers).
- [Documentation](https://www.tensorflow.org/?hl=pt-br) [Paper](https://dl.acm.org/doi/10.1145/1773912.1773922)
3. **Data Lakes and Data Lakehouse Architecture** – Evolution from data lakes to lakehouse, table formats (Delta Lake, Iceberg, Hudi).
- [Data Lakes Paper](http://sites.computer.org/debull/A15dec/p4.pdf) [Lakehouse Paper](https://people.eecs.berkeley.edu/~matei/papers/2020/vldb_delta_lake.pdf) [Table Formats Comparison](https://lakefs.io/hudi-iceberg-and-delta-lake-data-lake-table-formats-compared/)
4. **MLOps** – Machine Learning Operations, lifecycle management, deployment pipelines, monitoring.
- [Documentation](https://www.databricks.com/glossary/mlops) [Paper](https://arxiv.org/pdf/2205.02302)
5. **Samza** – Distributed stream processing framework, stateful computations, Kafka integration.
- [Documentation](https://samza.apache.org/) [Paper](http://sites.computer.org/debull/A15dec/p4.pdf)
6. **Beam** – Unified programming model for batch and streaming, portable across execution engines.
- [Documentation](https://beam.apache.org/) [Paper](https://static.googleusercontent.com/media/research.google.com/pt-BR//pubs/archive/43864.pdf)
7. **Dask** – Parallel computing library for Python, scalable analytics on large datasets.
- [Documentation](https://www.dask.org/) [Paper](https://pdfs.semanticscholar.org/73b5/8192f30bb6be8e798084d4481b97124570ed.pdf)
8. **Flink** – Stream processing engine, event-time processing, exactly-once semantics.
- [Documentation](https://flink.apache.org/) [Paper](https://github.com/manjunath5496/Apache-Flink-Papers/blob/main/a(2).pdf)
9. **Ray** – Distributed computing framework for AI/ML workloads, actor model, flexible task scheduling.
- [Documentation](https://www.ray.io/) [Paper](https://www.usenix.org/system/files/osdi18-moritz.pdf)
10. **Velox** – Unified execution engine for data processing, C++ library, columnar format optimization.
- [Documentation](https://velox-lib.io/) [Paper](https://vldb.org/pvldb/vol15/p3372-pedreira.pdf)
11. **TinkerPop** – Graph computing framework, Gremlin query language, vendor-agnostic graph abstraction.
- [Documentation](https://tinkerpop.apache.org/) [Paper](https://arxiv.org/pdf/1508.03843)
12. **NebulaGraph** – Distributed graph database, high-performance graph queries, scalable storage.
- [Documentation](https://www.nebula-graph.io/) [Paper](https://arxiv.org/pdf/1508.03843)
13. **Apache Polaris** – Open-source catalog for Apache Iceberg, multi-engine interoperability.
- [Documentation](https://polaris.apache.org/) [Paper](https://www.vldb.org/pvldb/vol13/p3461-goel.pdf)
14. **DuckDB** – In-process analytical database, OLAP for big data analytics.
- [Documentation](https://duckdb.org/) [Paper](https://dl.acm.org/doi/10.1145/3299869.3320212)
15. **DataFusion** – Extensible query execution framework in Rust, columnar processing.
- [Documentation](https://arrow.apache.org/datafusion/) [Paper](https://arrow.apache.org/papers/2017-Arrow-VLDB.pdf)
16. **Trino (formerly PrestoSQL)** – Distributed SQL query engine for big data.
- [Documentation](https://trino.io/) [Paper](https://trino.io/Presto_SQL_on_Everything.pdf)
17. **Druid** – Real-time analytics database, columnar storage, sub-second OLAP queries.
- [Documentation](https://druid.apache.org/) [Paper](https://arxiv.org/pdf/1909.07639)
18. **ClickHouse** – Column-oriented DBMS for real-time analytics, high-performance aggregations.
- [Documentation](https://clickhouse.com/docs/en/intro) [Paper](https://clickhouse.com/docs/en/about-us/distinctive-features)
19. **Pinot** – Real-time distributed OLAP datastore, low-latency analytics at scale.
- [Documentation](https://pinot.apache.org/) [Paper](https://engineering.linkedin.com/pinot/open-sourcing-pinot-scaling-wall)
20. **Recent conference papers**: Especially PhD students are encouraged to find scientific papers related to PDM in their areas of interest. Conferences such as SIGMOD, VLDB, CIDR, SoCC, and IEEE Big Data can be good sources.
